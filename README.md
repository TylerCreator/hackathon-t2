# hackathon-t2

## Описание задания

В рамках хакатона необходимо разработать приложение для сбора индивидуального графика работы сотрудников.

Согласно положению кейс-чемпионата, требуется реализовать информационную систему, в которой каждый сотрудник может войти в систему с рабочего компьютера и запланировать смены на будущие 2 недели. Руководитель подразделения должен иметь расширенный доступ и видеть нагрузку и распределение смен по сотрудникам и подразделению. Также в положении указано требование по формированию выгрузки в табличный формат с разбивкой по дням и указанием начала и конца смены :contentReference[oaicite:1]{index=1}

## Описание кейса

Приложение предназначено для автоматизации процесса планирования и учета рабочего графика сотрудников.

В системе должны быть реализованы два основных сценария использования:

- сотрудник входит в систему и формирует индивидуальный график работы;
- руководитель подразделения просматривает распределение смен, контролирует нагрузку и подтверждает соответствие плана и факта.

Шаблон `index.html` должен использоваться как основа пользовательского интерфейса приложения. 
[ссылка для просмотра шаблона](https://tylercreator.github.io/hackathon-t2/)

## Требования к приложению

### Базовые
- авторизация
- формирование индивидуального графика
- ролевая модель: админитсраторы, руководители, сотрудники и др.
- хранение данных о графике работы в базе данных
- визуализация графика по дням, неделям и месяцам
- публикация приложения на сервере

### Дополнительные
- интеграция с гугл таблицами
- подтверждение графика план/факт, запись информации в бд
- отчеты по времени план/факт
- возможность работы с несколькими организациями
- возможность работы с несколькими командами внутри одной организации, учитывая вложенность

### участники имеют право не соблюдать требования, реализовавывать свои идеи и функции, чтобы создать лучшее приложение для планирования индивидуального графика сотрудников
